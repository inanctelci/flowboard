## E2E Flow Walks (4/4)

### Flow 1 — Character generation (PASS)
1. User clicks Generate on a character node → opens GenerationDialog
2. `GenerationDialog.tsx:809` renders `<CharacterWizard rfId={rfId} onDone={closeGenerationDialog} />`
3. CharacterWizard lazy-reads `useBoardStore.getState().nodes.find(n => n.id === rfId)?.data` to seed all 12 char fields (685d810 fix)
4. User navigates step tabs (no hard gates); edits chip rows / free-text
5. Review step calls `buildCharacterPrompt(config)` → renders prompt preview in `<pre>`
6. Generate button: `toCharacterDataPatch(config, prevData)` → `patchNode(dbId, {data: delta})` → `dispatchGeneration(rfId, opts)` → `onDone()` calls `closeGenerationDialog`

### Flow 2 — Preset save/load (PASS)
1. Review step: user types name → click Save → `addPreset(name, config)`
2. `characterPresets.ts` checks `length >= 50` → if OK, generates id (`crypto.randomUUID()`), appends, calls `persist(updated)` → `localStorage.setItem("flowboard.character.presets.v1", JSON.stringify({version: 1, data: presets}))`
3. Reload page → `loadPersisted()` reads localStorage → `PersistedPresetsSchema.safeParse(JSON.parse(raw))` → if success populate state, if fail route error key to Toaster
4. PresetList renders saved presets → click → `onLoad({...preset.config})` → CharacterWizard updates config + jumps to step 5 (Review)

### Flow 3 — v1.0 board compatibility (PASS — static analysis)
1. v1.0 board node persisted with `{ charCountry: "vn", charVibe: "clean" }` (no charEthnicity)
2. Board hydration (any of 3 sites) calls `migrateCharacterNodeData(data)` → inlined `LEGACY_COUNTRY_TO_ETHNICITY["vn"]` returns "Vietnamese" → data becomes `{ charCountry: "vn", charVibe: "clean", charEthnicity: "Vietnamese" }`
3. ResultViewer renders Vietnamese pill via `ethnicityLabel(t, "Vietnamese", "vn")` (free-text path)
4. User opens GenerationDialog → wizard seeds `{ charVibe: "clean", charEthnicity: "Vietnamese" }` (charCountry not seeded; new data model only)

### Flow 4 — Error handling (PASS)
1. 51st preset save: `addPreset` early-returns with `set({error: "wizard.error.libraryFull"})` (i18n key path)
2. Toaster's `useCharacterPresetsStore((s) => s.error)` picks up the key → `t(errorKey)` translates → renders
3. localStorage quota exception during `persist`: caught, returns `"wizard.error.storageFull"` key → same Toaster pipeline
4. Corrupt localStorage JSON: `loadPersisted()` `safeParse` fails → returns `{presets: [], error: "wizard.error.presetsCorrupted"}` → store initialized empty, Toaster shows message
