# Milestone v1.1: Character Creation Rework

**Status:** ✅ SHIPPED 2026-06-17
**Phases:** 5–7
**Total Plans:** 3 plans (one per phase, coarse granularity)

## Overview

Replaced Flowboard's hardcoded character preset system (`CHARACTER_GENDERS` / `CHARACTER_COUNTRIES` / `CHARACTER_VIBES` — 2 genders, 7 Vietnamese-labelled nationalities, 6 baked-token vibes) with a guided multi-step wizard producing structured, reusable character configurations.

The new architecture:
- `frontend/src/lib/character/` — Zod schema (`CharacterConfigSchema`), pure prompt assembler (`buildCharacterPrompt`), delta-only patch helper (`toCharacterDataPatch`), convert-on-read migration (`migrateCharacterNodeData`), and inlined vibe tokens (`vibeTokens.ts`)
- `frontend/src/store/characterPresets.ts` — hand-rolled localStorage Zustand slice for the named-preset library (key `flowboard.character.presets.v1`, version 1)
- `frontend/src/components/character/` — 5-step `CharacterWizard` + 5 step components + `PresetList`, mounted inside the existing `GenerationDialog` `{isCharacter && (...)}` block
- `frontend/src/components/ResultViewer.tsx` — pills updated to render new structured fields with `LEGACY_COUNTRY_LABELS` shim for unmigrated v1.0 nodes

Zero backend changes. Zero new test runner. v1.0 boards keep loading via three migrated hydration sites (`loadInitialBoard`, `switchBoard`, `refreshBoardState`).

**Key Decisions:**

| Decision | Rationale |
|----------|-----------|
| Flat `char*` keys on node.data (no nested `character: {}` object) | Shallow-merge PATCH contract requires top-level keys; nested object would be wholesale-replaced |
| Wizard mounts inside GenerationDialog | Reuse existing modal infra (backdrop, focus trap, ESC); no second modal layer |
| Wizard transient state in useState (not Zustand slice) | Discarded on cancel; not persisted mid-step (WIZARD-05) |
| Preset library in localStorage via hand-rolled persistence | Matches `references.ts` codebase convention; Reference table is media-first (wrong semantic for config blobs) — `zustand/middleware/persist` exists but unused in repo |
| Convert-on-read migration (charCountry → charEthnicity) — no startup PATCHes | Thundering herd of PATCHes on load creates a false "board modified" signal; lazy migration is the correct pattern |
| Zod 4 added as the only new runtime dependency | Runtime parse of localStorage CharacterConfig blobs; prevents corrupt state from blocking the app |
| Stable English keys / English prose stored on node.data — never translated labels | Boards must be locale-independent; assembler stays correct on any installed locale |
| Closed-enum option keys via constant lookup tables (no dynamic t() construction) | Honors v1.0 i18n discipline (BUGS-02 class regression prevention) |
| WIZARD-05 honored literally — discard-on-cancel | UI-SPEC proposed draft preservation across reopen; CONTEXT.md ruled this out in favor of requirement clarity |
| Coarse granularity: 3 phases (research recommended 5) | Phase 3 (preset store) merged into Phase 6 with wizard UI; Phase 4 (constants removal) merged into Phase 7 with i18n audit |
