## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user-supplied localStorage → i18n init | The `flowboard.i18n.locale` value is read at module load. A malicious or corrupted value could attempt to inject unexpected language codes. |
| npm registry → frontend bundle | Three new runtime packages enter the production bundle. Supply-chain compromise of any of them executes in the user's browser. |
| backend error tokens → humanizeBackendError | Tokens originate in the agent/worker and are passed through case-folding then string matching. A maliciously crafted token cannot escape the function (returns null on miss), but locale-sensitive case folding is the bug we are fixing. |
| user-typed content (data.title, data.prompt, ref.label) → JSX | This phase does NOT wrap these in `t()` — they render raw via React, which already HTML-escapes. The phase explicitly preserves this boundary; Phase 2 EXTRACT-07 enforces it permanently. |
