<objective>
Extract every translatable string in the `frontend/src/canvas/` directory into the i18n catalog. This is the largest and highest-risk plan in Phase 2 — `NodeCard.tsx` alone has 52 translatable rows and renders user-authored data (`data.title`) immediately adjacent to UI chrome. The user-data boundary must be respected at every site.

Purpose: Satisfy EXTRACT-01 (JSX text nodes), EXTRACT-02 (JSX attribute props), EXTRACT-06 (brand names stay out of catalog), and EXTRACT-07 (user-authored content never wrapped) for the canvas surface.

Output:
- All canvas `.tsx` files use `useTranslation()` + `t("node.*"|"palette.*")` instead of hardcoded English
- `frontend/src/i18n/locales/en.json` gains ~55 new keys (node.*, palette.*)
- `frontend/src/i18n/locales/tr.json` gains the same ~55 keys as empty-string stubs
- `npm run lint` exits 0
</objective>

1. NEVER delete an existing key (Phase 1 keys `time.*` stay).
2. NEVER reorder existing keys.
3. APPEND new keys alphabetically within the `node.*` and `palette.*` area prefixes.
4. Keep one key per line, JSON-valid, no trailing commas — the merge is textual.
