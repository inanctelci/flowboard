<headless_rule_reminder>
Per CONTEXT.md §Headless translation rule (CRITICAL):

```ts
// In activity-meta.ts (NOT a React component):
import i18n from "../../i18n/i18n";
const label = i18n.t("activity.gen_image");
```

```tsx
// In ActivityRow.tsx (React component):
const { t } = useTranslation();
return <span>{t("activity.row_cancel")}</span>;
```

NEVER call `useTranslation()` in `activity-meta.ts`. The typed-key TypeScript augmentation will NOT catch this — but it will throw at runtime as a hooks violation.
</headless_rule_reminder>

    **For the four .tsx files** (ActivityBell, ActivityDropdown, ActivityRow, ActivityDetailModal):

    Walk inventory rows; for each translatable row append the `Key proposal` (under `activity.*` prefix) to en.json with the English value, then wrap the source site with `t("activity.<concept>")`. These files have ~7 total translatable rows across them — straightforward.

    **For activity-meta.ts** (the harder one — ~15 rows):

    The file currently defines TWO tables (`ACTIVITY_TYPE_META` and `STATUS_META`) and TWO utility functions (`relativeTime()` and `formatDuration()`). The inventory lists 8 type labels, 6 status labels, 5 relativeTime returns, and 3 formatDuration returns.

    **Decision — table-label pattern**: change the `label` field on each `ACTIVITY_TYPE_META` / `STATUS_META` entry from a literal string (`"Auto-Prompt"`) to either:
    - **Option A (lazy)**: store a key string (`"activity.auto_prompt"`) and require callers to wrap with `i18n.t()`. This shifts the burden to call sites and is verbose at every call. Reject this — too invasive across consumers.
    - **Option B (eager — RECOMMENDED)**: change the `label` field to a getter that returns `i18n.t("activity.auto_prompt")`. This works because `i18n` is a module-level singleton; every access to `.label` re-evaluates against the current locale.

    Concrete pattern for Option B (use this):

    ```ts
    // Before (current):
    export const ACTIVITY_TYPE_META: Record<ActivityType, { label: string; ... }> = {
      auto_prompt: { label: "Auto-Prompt", ... },
      ...
    };

    // After:
    export const ACTIVITY_TYPE_META: Record<ActivityType, { get label(): string; ... }> = {
      auto_prompt: { get label() { return i18n.t("activity.auto_prompt"); }, ... },
      ...
    };
    ```

    Alternative if getters complicate TypeScript inference: switch to a helper function `activityTypeLabel(type: ActivityType): string` that callers invoke (`activityTypeLabel(item.type)` instead of `ACTIVITY_TYPE_META[item.type].label`). Pick whichever produces fewer changes at call sites — both are valid. Document the choice in SUMMARY.

    Apply the same pattern to STATUS_META labels.

    **`relativeTime()` utility function**: per inventory rows 52-60, replace the return values:

    ```ts
    // Before:
    if (sec < 5) return "just now";
    if (sec < 60) return `${sec}s ago`;
    ...

    // After:
    if (sec < 5) return i18n.t("activity.just_now");
    if (sec < 60) return i18n.t("activity.sec_ago", { count: sec });
    if (min < 60) return i18n.t("activity.min_ago", { count: min });
    if (hr < 24) return i18n.t("activity.hr_ago", { count: hr });
    return i18n.t("activity.day_ago", { count: day });
    ```

    Use `_one`/`_other` plural suffix keys for the `{{count}}` returns per Phase 1's `time.*` precedent:
    - `activity.sec_ago_one`: "{{count}}s ago", `activity.sec_ago_other`: "{{count}}s ago"
    - `activity.min_ago_one`: "{{count}}m ago", `activity.min_ago_other`: "{{count}}m ago"
    - `activity.hr_ago_one`: "{{count}}h ago", `activity.hr_ago_other`: "{{count}}h ago"
    - `activity.day_ago_one`: "{{count}}d ago", `activity.day_ago_other`: "{{count}}d ago"
    - `activity.just_now`: "just now" (no plural — bare key)

    Call signature stays `i18n.t("activity.sec_ago", { count: sec })` — i18next picks `_one` vs `_other` based on the count.

    **`formatDuration()` strings** (`"${ms}ms"`, `"${s}s"`, `"${min}m ${s}s"`): per CONTEXT.md §Ambiguity #4, DO NOT translate. These are SI abbreviations universal across locales. Add a one-line comment `// i18n: do-not-translate (technical — SI duration abbreviations)` above the function. Leave the function body untouched.

    **Unicode glyph icons** (`⋯`, `⟳`, `✓`, `✗`, `⊘`, `⏱`): per inventory's do-not-translate flags — these are pictographic indicators in the same enum entries as the labels. They stay verbatim. They live in the `icon` field of each meta entry (or wherever the file places them). The conversion of `label` to a getter does NOT affect the icon field — leave icons as plain strings.

    Run `cd frontend && npm run lint` after each file. Commits per CONTEXT.md §Atomicity: one for the 4 .tsx files (or per-file), one for activity-meta.ts, ride en.json with whichever file introduces the keys.
